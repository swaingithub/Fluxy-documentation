# Fluxy Master Feature Catalog

This is the Ultimate Index of Fluxy. Every single tool in the framework is listed here with its purpose and where to find detailed documentation.

---

## 1. State & Reactivity (Fluxy Core)
The fundamental building blocks for your application's logic.

| Feature | Type | Purpose | Documentation |
| :--- | :--- | :--- | :--- |
| **`flux(initial)`** | Signal | Define a reactive state unit. | [State Basics](/docs/fundamentals/state-management) |
| **`persist: true`** | Modifier | Automatically save/load signal state to storage. | [Persistence](/docs/fundamentals/state-management#persistence) |
| **`computed(() => ...)`** | Signal | Derived state that cached and updates automatically. | [Reactivity](/docs/fundamentals/reactive-engine) |
| **`FluxEffect(() => ...)`**| Effect | Run side-effects (logging, API calls) when state changes. | [Effects](/docs/fundamentals/core-concepts#effects) |
| **`batch(() => ...)`** | Utility | Group multiple state updates into one UI frame. | [Performance](/docs/fundamentals/performance) |
| **`fluxSelector(obj, key)`**| Signal | Observe a specific property of a complex object. | [Selectors](/docs/fundamentals/state-management#selectors) |
| **`fluxWorker(() => ...)`**| Utility | Run heavy logic in background micro-tasks. | [Workers](/docs/fundamentals/architecture) |
| **`fluxField(label)`** | Signal | Specialized signal for form inputs with validation. | [Forms](/docs/forms) |
| **`fluxList([])`** | Collection| Reactive list with specialized update methods. | [Advanced Control](/docs/fundamentals/advanced-control#reactive-lists) |
| **`fluxHistory(val)`**| Collection| State with built-in Undo/Redo capability. | [Advanced Control](/docs/fundamentals/advanced-control#state-history-undo--redo) |
| **`.tr` / `.trText()`**| Signal | Reactive internationalization extension. | [i18n Guide](/docs/fundamentals/i18n) |

| **`Fx.col() / Fx.row()`**| Layout flexboxes with `.gap()` and `.align()`. | [Flex Layout](/docs/layout#flex) |
| **`Fx.layout()`** | Multi-device switcher (Mobile/Tablet/Desktop). | [Responsive](/docs/layout#responsive) |
| **`Fx.reveal()`** | Staggered entrance animations for content. | [Animations](/docs/style#animations) |
| **`Fx.badge()`** | Standard reactive notification badge. | [Components](/docs/components/feedback) |
| **`Fx.icon()`** | Simplified icon handler with reactive color/size. | [Icons](/docs/style#icons) |
| **`Fx.loader()`** | Global industrial loading indicators. | [Feedback](/docs/components/feedback) |
| **`Fx.toast`** | `success()`, `error()`, `info()` global alerts. | [Feedback](/docs/components/feedback#toast) |
| **`Fx.field()`** | Self-binding input field (No controllers needed). | [Forms](/docs/forms) |

## 5. Global Registry & DI
How to manage the lifecycle of your application.

| Utility | Purpose |
| :--- | :--- |
| **`Fluxy.init()`** | Bootstraps the framework in main.dart. |
| **`Fluxy.register(T)`**| Register a Controller/Plugin to the global scope. |
| **`Fluxy.use<T>()`** | Retrieve a registered module from anywhere. |
| **`Fluxy.autoRegister()`**| Scans the app for standard Fluxy modules. |
| **`Fx.platform`** | Unified access to all connected platform plugins. |
| **`fluxy cloud`** | CLI Engine | Scaffold GitHub Actions for Android/iOS. | [Cloud Guide](/docs/production/cloud) |
