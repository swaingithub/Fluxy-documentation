# Atomic Styling DSL

Fluxy provides a powerful Atomic Styling DSL (Domain-Specific Language) via the `Fx` namespace and widget extensions. This allows you to build complex UI hierarchies using chainable modifiers, drastically reducing nesting and boilerplate.

## Syntax: Chainable Modifiers

Every Fluxy widget (accessible via `Fx`) and even standard Flutter widgets (via `FluxyWidgetExtension`) can be styled using chainable methods.

```dart
Fx.box()
  .p(16)            // Shorthand for padding
  .bg(Colors.blue)  // Background color
  .rounded(12)      // Border radius
  .shadowXL()       // Large shadow token
  .render();
```

## Proxy-based DSL (New in 0.1.4)

Fluxy v0.1.4 introduces a more expressive **Proxy-based DSL** that allows you to access style properties as fields rather than methods for common tokens. This results in even cleaner, more readable code.

```dart
Fx.text("Hello Fluxy")
  .font.lg.bold()      // New Property Proxies
  .color.blue500    // Proxy syntax for color
  .shadow.md()      // Shadow proxy
  .mt(8);           // Traditional shorthand still available
```

### Why use Proxy DSL?
1. **Readability**: It reads more like a declaration than a series of function calls.
2. **Type Safety**: Proxies are specifically mapped to framework tokens, ensuring you use valid colors and weights.
3. **Speed**: Autocomplete works seamlessly with proxy getters.

## Shorthand Pairs

Fluxy uses intuitive shorthand pairs for common layout properties:

### Padding & Margin
Fluxy is designed to be flexible. You can use full descriptive names or rapid-fire shorthands interchangeably:
- **Padding**: `.padding()`, `.p()`, or `.pad()`
- **Horizontal/Vertical**: `.px()`, `.py()`, `.paddingX()`, `.paddingY()`
- **Directional**: `.pt()` (top), `.pb()` (bottom), `.pl()` (left), `.pr()` (right)
- **Margin**: `.margin()`, `.m()`, `.mt()`, `.mb()`, `.ml()`, `.mr()`

### Size Aliases
- **Dimensions**: `.w()` or `.width()`, `.h()` or `.height()`
- **Bulk**: `.size(w, h)`, `.square(size)`
- **Full Scale**: `.fullWidth()`, `.fullHeight()`, or shorthands `.wFull()`, `.hFull()`

### Styling & Colors
- **Background**: `.bg(color)` or `.background(color)`
- **Radius**: `.rounded(radius)` or `.radius(radius)`
- **Shadows**: `.shadow(token)` or quick presets like `.shadowXL()`
- **Modern Colors**: Always use `.withValues(alpha: 0.x)` for transparency (modern API).

```dart
Fx.box()
  .square(100)
  .bg(Colors.red.withValues(alpha: 0.5))
  .rounded(FxTokens.radius.md)
  .render();
```

## Interactions & Animations

Fluxy makes it easy to add dynamic interactions and smooth transitions directly in the style chain:

- `.onHover( (s) => s.scale(1.1) )`: Apply styles when the mouse hovers over the widget.
- `.onPressed( (s) => s.op(0.8) )`: Apply styles when the widget is pressed.
- **Animations**: Use `.animate()` to implicitly animate any style changes (background, size, opacity, etc.).

```dart
Fx.box()
  .square(100)
  .bg(Colors.blue)
  .animate(duration: 500.ms, curve: Curves.easeOut)
  .onHover((s) => s.bg(Colors.blueAccent).scale(1.1))
  .render();
```

## Conditional Visibility

You can conditionally show or hide widgets without complex `if` statements in your build method using the `.show()` and `.hide()` modifiers.

```dart
Fx.text("Conditional Text")
  .show(isLogged.value)
  .render();
```

## Layouts & Flex Control

Fluxy v0.1.3 introduces powerful new modifiers for `Fx.box` and linear layouts (`Fx.row`, `Fx.column`) for cleaner responsive designs.

### Linear Layouts

Standard linear layouts use the same fluent philosophy:

```dart
Fx.column()
  .gap(16) // Built-in spacing between children
  .crossStart()
  .children([
    Fx.text("Header").h2().render(),
    Fx.text("Body text here...").render(),
  ])
  .render();
```

### Fluent Flex Modifiers (New in 0.1.4)

Fluxy now provides comprehensive flex layout modifiers to control alignments and spacing fluently, just like in Tailwind CSS. You can use these modifiers on any `Fx.row`, `Fx.col`, or even a generic `Fx.box`.

#### Alignment & Justification
- `.justifyStart()`, `.justifyCenter()`, `.justifyEnd()`
- `.justifyBetween()`, `.justifyAround()`, `.justifyEvenly()`
- `.itemsStart()`, `.itemsCenter()`, `.itemsEnd()`, `.itemsStretch()`
- **New Shortcut**: `.alignCenter()` (Centers both axes)

#### Spacing
- `.gap(double)`: Adds equal spacing between all children.

### Usage Examples

Here is how you can use them to create beautiful layouts with zero effort:

```dart
// A centered row with items spread out and a 20px gap
Fx.row(
  children: [
    Fx.text("Left"),
    Fx.text("Middle"),
    Fx.text("Right"),
  ],
)
.justifyBetween()
.itemsCenter()
.gap(20)
.p(16);

// A vertical column centered both ways
Fx.column(
  children: [
    Fx.avatar(image: "URL"),
    Fx.text("User Name").bold(),
  ],
)
.justifyCenter()
.itemsCenter()
.gap(12)
.fullWidth();
```

#### Why use these instead of constructor parameters?

While `Fx.row` and `Fx.col` still support `mainAxisAlignment` and `crossAxisAlignment` in their constructors, using these modifiers allows you to chain layout properties with styles (padding, background, etc.) in a single, readable flow. It also makes your code feel much more like modern CSS/Tailwind.

```dart
Fx.box()
  .p(20)
  .asRow()       // Determine direction
  .gap(12)       // Gap between children
  .justifyBetween()
  .itemsCenter()
  .children([
    Fx.text("Left Item"),
    Fx.text("Right Item"),
  ])
  .render();
```

## Context Extensions

Fluxy v0.1.4 extends `BuildContext` with helpful properties to reduce boilerplate when accessing theme and screen information.

- `context.theme`: Access the current `ThemeData`.
- `context.colors`: Shortcut to `Theme.of(context).colorScheme`.
- `context.isDark`: Boolean check for dark mode.
- `context.screenSize`: Access `MediaQuery.of(context).size`.

```dart
Fx.box()
  .bg(context.colors.primary)
  .w(context.screenSize.width * 0.8)
  .child(
    Fx.text(context.isDark ? "Dark Side" : "Light Side").render()
  )
  .render();
```
